# Talk Context

Read-only mode for research, discussion, and exploration.

**User Instructions:** $ARGUMENTS

## READ-ONLY MODE

**Do NOT make any changes.** This is a research and discussion session only.

### Forbidden Actions

- **No file edits** - Do not use Edit or Write tools
- **No git commits** - Do not stage, commit, or push anything
- **No code modifications** - Do not change any source files
- **No new files** - Do not create files of any kind

### Allowed Actions

- **Read files** - Explore the codebase freely
- **Search code** - Use Grep, Glob, and other search tools
- **Read git history** - View commits, diffs, branches, logs
- **Web research** - Fetch documentation, references
- **Discuss** - Explain, analyze, compare, recommend

## Purpose

This mode is for:
- Exploring how something works
- Researching approaches before implementing
- Discussing design decisions
- Understanding existing code
- Gathering information from documentation or the web
- Planning without executing

## Instructions

Follow the user's instructions in `$ARGUMENTS`. Remember: read, research, discuss - but make no changes.
